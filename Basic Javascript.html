(4) What are undeclared and undefined variables ?

=> Undeclared Variables: An undeclared variable is a variable that has been used in code without being formally declared using a 'var', 'let', or 'const' keyword.

=> When a variable is used without declaration, JavaScript will implicitly create a global variable in non-strict mode, which might lead to unexpected behavior and bugs in your code.

=> Undefined Variables: An undefined variable is a variable that has been declared but not assigned any value. In JavaScript, if a variable is declared but not initialized, it holds the special value 'undefined; by default.




(5) What is the difference between ViewState and SessionState ?

=> ViewState: ViewState is a client-side state management technique used in ASP.NET web forms. It is designed to preserve the state of a server-side control or page across postbacks. It stores the state of the page or control in a hidden field on the page itself. This hidden field is sent back and forth between the server and the client with each request and response. ViewState is specific to a single page and maintains the state of controls or data within that particular page. It is useful for retaining information about controls' state between round-trips to the server.

=> SessionState: SessionState, on the other hand, is a server-side state management technique used to store user-specific information across multiple pages or requests for a particular user session. It stores data on the server associated with a unique session ID for each user. This data persists throughout the user's interaction with the web application until the session ends (e.g., when the user logs out or after a period of inactivity). SessionState is not limited to a single page but is accessible across multiple pages within the same session. It can store user-specific information like user preferences, shopping cart items, user authentication details.

=> Scope: ViewState maintains state information for a specific web page, while SessionState retains data across multiple pages for a user session.

=> Storage Location: ViewState data is stored as a hidden field on the web page itself, whereas SessionState data is stored on the server.

=> Size and Performance: ViewState tends to increase the size of the web page, affecting performance for larger amounts of data.




(6) What is === operator ?

=> In JavaScript, the === operator is known as the "strict equality operator."

=> It is used for comparing two values for equality without performing type coercion.

=> It checks both the value and the data type of the operands being compared.

=> Example : console.log(5 === 5);
console.log(5 === '5');
console.log(true === true);
console.log(true === 1);
console.log('hello' === 'hello');
console.log(null === undefined);
console.log(0 === -0);
console.log(NaN === NaN);

 (9) What are all the looping structures in JavaScript ?
    
    
 => For Loop : The for loop is used to execute a block of code a specified number of times.
  
    
  Example :  // Using a for loop to print numbers from 1 to 5 

        for (let i = 1; i <= 5; i++) 

        {
          console.log(i);
        }  
  
    
 => while loop: The 'while loop' is used to execute a block of code as long as a specified condition is true.
    
    
 Example : let i = 0;  

        while (i < 5) { 

          console.log(i);  

          i++; 

        }

    
 => do...while loop : The 'do...while' loop is similar to the while loop, but it executes the code block once before checking the condition.
  
    
 Example : let i = 0; 

        do {   
 
          console.log(i);  

          i++;  

        } while (i < 5);

 Example : onst person = {  

        name: 'John',  

        age: 30,   

        gender: 'male'  

      };  

      
      for (let key in person) {  

        console.log(key + ': ' + person[key]);  

      }

    
 => for...of loop : The 'for...of' loop iterates over iterable objects like arrays, strings, maps, sets, etc., accessing their values directly.

    
 Example : const fruits = ['apple', 'banana', 'orange'];

 (12) What are all the types of Pop up boxes available in JavaScript ?
  
    
 => Alert Box (alert()):

        The 'alert()'' method displays a simple dialog box with a message and an "OK" button. 

        It is commonly used to show an informative message or to get the user's attention.
  
    
 => Confirm Box (confirm()):

        The 'confirm()'' method displays a dialog box with a message and two buttons: "OK" and "Cancel." 

        It is used to ask the user for confirmation before proceeding.
    
    
 => Prompt Box (prompt()):

        The 'prompt()' method displays a dialog box with a message, an input field for the user to enter data, and "OK" and "Cancel" buttons. 

        It is used to prompt the user to enter some information.

 (15) What are the disadvantages of using innerHTML in JavaScript ?

    
 => Security Risks (Cross-Site Scripting - XSS): Assigning strings directly to 'innerHTML' that contain user-generated or untrusted content can pose security risks such as Cross-Site Scripting (XSS). If the content includes malicious scripts, they might be executed when setting 'innerHTML', potentially compromising the security of the web application.

    
 => String Parsing and Serialization: The process of converting strings to DOM elements and vice versa when using 'innerHTML' can lead to parsing and serialization overheads, especially when dealing with complex or nested HTML structures.

    
 => ompatibility and Browser Quirks: The behavior of 'innerHTML' might vary slightly across different browsers and might have certain quirks or inconsistencies, potentially causing unexpected behavior in some cases.
